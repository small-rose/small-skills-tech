## 引用格式规范

### 标准引用
```markdown
**来源**: https://example.com/documentation
**验证状态**: ✅ 已验证（2026-06-18）
**可信度**: ⭐⭐⭐⭐⭐ 官方文档
**来源类型**: 官方文档
```

### 未验证引用
```markdown
**来源**: https://example.com/blog
**验证状态**: ❌ 未验证
**失败原因**: 404 页面不存在
**可信度**: ⭐⭐ 社区博客
**来源类型**: 个人博客

> ⚠️ **未经过验证**: 此解决方案来自社区讨论，尚未在官方文档中确认。
> 请在测试环境中验证后再应用到生产环境。
```

### 引用优先级
1. 官方文档（⭐⭐⭐⭐⭐）
2. 官方 GitHub（⭐⭐⭐⭐⭐）
3. 权威社区（Stack Overflow、GitHub Issues）（⭐⭐⭐⭐）
4. 官方博客（⭐⭐⭐⭐）
5. 技术书籍（⭐⭐⭐）
6. 个人博客（⭐⭐）
7. AI 生成（⭐）

## 问题分类标准

### 问题类型
| 类型 | 说明 | 示例 |
|------|------|------|
| 配置问题 | 配置项错误或缺失 | application.yml 配置错误 |
| 依赖问题 | 依赖冲突或缺失 | 版本不兼容 |
| 运行时错误 | 程序运行时抛出异常 | NullPointerException |
| 性能问题 | 程序运行缓慢或资源占用高 | 内存溢出、CPU 占用高 |
| 部署问题 | 部署过程中遇到的问题 | Docker 构建失败 |
| 环境问题 | 开发环境配置问题 | JDK 版本不兼容 |

### 严重程度
| 程度 | 说明 | 响应时间 |
|------|------|---------|
| 高 | 阻塞核心功能、生产环境故障 | 立即 |
| 中 | 影响部分功能、有临时解决方案 | 1-2 小时 |
| 低 | 轻微影响、有明确解决方案 | 1 天内 |
