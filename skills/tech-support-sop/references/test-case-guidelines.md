# 测试用例指南

## 测试评估维度

### 复杂度评估

| 复杂度 | 说明 | 处理方式 |
|--------|------|---------|
| **低** | 一键命令，无需额外配置 | 直接提供测试脚本 |
| **中** | 需要简单配置或环境准备 | 询问用户环境，提供方案 |
| **高** | 需要复杂环境搭建或特殊权限 | 详细询问环境，提供多个方案 |

### 评估标准

| 维度 | 低复杂度 | 中复杂度 | 高复杂度 |
|------|---------|---------|---------|
| 命令数量 | 1-3 条 | 4-10 条 | 10+ 条 |
| 配置文件 | 无需修改 | 修改 1-2 个文件 | 修改多个文件 |
| 环境要求 | 通用环境 | 特定版本 | 特殊环境/权限 |
| 依赖安装 | 无 | 少量 | 大量 |
| 执行时间 | <1 分钟 | 1-5 分钟 | 5+ 分钟 |
| 风险程度 | 低 | 中 | 高 |

## 测试方案设计

### 方案 A: 自动测试（低复杂度）

适用于：一键命令、配置验证、简单修复

**特点**：
- 用户无需手动操作
- 自动执行并报告结果
- 失败时自动回滚

**示例**：
```bash
#!/bin/bash
# 自动测试脚本示例

echo "=== 测试开始 ==="

# 1. 环境检查
echo "--- 环境检查 ---"
if ! command -v java &> /dev/null; then
    echo "❌ Java 未安装"
    exit 1
fi
echo "✅ Java 版本: $(java -version 2>&1 | head -1)"

# 2. 执行修复
echo "--- 执行修复 ---"
# 修复命令...

# 3. 验证修复
echo "--- 验证修复 ---"
if [验证条件]; then
    echo "✅ 测试通过"
    exit 0
else
    echo "❌ 测试失败"
    exit 1
fi
```

### 方案 B: 半自动测试（中复杂度）

适用于：需要用户确认环境、分步执行

**特点**：
- 需要用户提供环境信息
- 分步执行，每步确认
- 提供详细说明

**示例**：
```markdown
## 测试步骤

### 步骤 1: 环境检查
请执行以下命令并告诉我结果：

```bash
java -version
mvn -version
```

### 步骤 2: 配置修改
根据步骤 1 的结果，修改以下配置文件：
- 文件: `pom.xml`
- 修改: 将 Java 版本改为 [版本]

### 步骤 3: 验证
执行以下命令验证：
```bash
mvn clean compile
```

请告诉我执行结果。
```

### 方案 C: 手动测试（高复杂度）

适用于：复杂环境搭建、需要特殊权限

**特点**：
- 详细询问环境信息
- 提供多个备选方案
- 建议用户在测试环境执行

**示例**：
```markdown
## 环境要求

- **操作系统**: [如 Windows 10、Ubuntu 22.04]
- **JDK 版本**: [如 JDK 17、JDK 21]
- **Maven 版本**: [如 Maven 3.8.6]

- **网络连接**: [需要访问 Maven 中央仓库]

## 测试方案

### 方案 1: 本地测试
1. [步骤 1]
2. [步骤 2]
3. [步骤 3]

### 方案 2: Docker 测试
1. [步骤 1]
2. [步骤 2]
3. [步骤 3]

## 注意事项

- ⚠️ 请在测试环境执行
- ⚠️ 建议备份重要数据
- ⚠️ 执行前请确认网络连接
```

## 验证命令模板

### Java 验证命令

```bash
# 环境检查
java -version
javac -version
mvn -version
gradle -version

# 项目检查
cd project-dir
mvn clean compile
gradle build

# 依赖检查
mvn dependency:tree
gradle dependencies

# 测试执行
mvn test
gradle test
```

### Python 验证命令

```bash
# 环境检查
python --version
python3 --version
pip --version

# 项目检查
cd project-dir
pip install -r requirements.txt
pip install -e .

# 依赖检查
pip list
pip freeze

# 测试执行
python -m pytest
python -m unittest
```

### Nginx 验证命令

```bash
# 环境检查
nginx -v
nginx -t

# 配置检查
nginx -t -c /path/to/nginx.conf

# 服务管理
systemctl status nginx
systemctl restart nginx

# 日志检查
tail -f /var/log/nginx/error.log
```

### Node.js 验证命令

```bash
# 环境检查
node --version
npm --version
yarn --version

# 项目检查
cd project-dir
npm install
yarn install

# 依赖检查
npm list
yarn list

# 测试执行
npm test
yarn test
```

### Docker 验证命令

```bash
# 环境检查
docker --version
docker-compose --version

# 容器检查
docker ps
docker images

# 服务管理
docker-compose up -d
docker-compose down

# 日志检查
docker-compose logs -f
```

## 测试建议

### 最佳实践

1. **备份数据**: 测试前备份重要数据
2. **测试环境**: 尽量在测试环境执行
3. **分步执行**: 复杂测试分步执行
4. **记录结果**: 详细记录测试结果
5. **及时回滚**: 发现问题及时回滚

### 常见问题

| 问题 | 处理方式 |
|------|---------|
| 测试环境与生产环境不一致 | 使用 Docker 或虚拟机创建一致环境 |
| 测试数据不足 | 使用模拟数据或测试数据生成工具 |
| 测试时间过长 | 优化测试用例，减少不必要的步骤 |
| 测试结果不稳定 | 分析原因，优化测试用例 |
